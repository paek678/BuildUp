# 졸업 프로젝트 결과 보고서

**작품명**: 테네브리스(Tenebris) / Arena Combat

---

## 1. 목적

본 작품은 Unity 기반 2인 협동 보스전 3D 탑다운 멀티플레이 액션 게임을 직접 기획·구현하되, 핵심 목적은 **강화학습 기반 적응형 보스 AI** 구조를 설계하는 데 있다.

기존 보스전 AI는 상태머신이나 규칙 기반 패턴으로 구현되어, 플레이어가 패턴을 암기하면 긴장감이 급격히 하락한다. 특히 2인 협동 구조에서는 두 플레이어의 위치 관계, 공격 성향, 패링 빈도, 로프 활용 방식 등 다양한 변수가 전투 양상에 큰 영향을 주기 때문에, 단순 규칙 기반 AI만으로는 복합적인 전투 흐름을 충분히 반영하기 어렵다.

이를 해결하기 위해 본 프로젝트는 ML-Agents PPO 강화학습을 활용하여 보스가 플레이어의 누적 행동 데이터를 기반으로 전술적 대응을 선택하도록 설계하였다. 네트워크 구조는 Host-authoritative 방식을 적용하여 서버가 모든 판정의 최종 권한을 가지도록 하였고, 그 위에 강화학습 기반의 상위 판단 구조를 얹을 수 있도록 아키텍처를 구성하였다.

## 2. 주요 개발 내용

### 2-1. 하이브리드 보스 AI 구조

보스 AI의 모든 행동을 학습시키는 것은 비효율적이며 안정성 문제가 크다. 따라서 본 프로젝트는 계층을 분리하는 하이브리드 구조를 채택하였다.

| 계층 | 담당 영역 | 방식 |
|------|----------|------|
| **상위 의사결정** | 위치 선정, 압박 대상 선택, 거리 조절, 분산 유도 | **ML-Agents PPO 강화학습** |
| 상태 전환 | Idle → Approach → Attack → Cooldown 순환, HP% 페이즈 전환 | FSM 규칙 기반 |
| 스킬 발동 | 상황별 스킬 선택, 쿨타임 관리 | 가중치 규칙 + 편향 점수 |

이 구조는 강화학습의 유연성과 규칙 기반 시스템의 안정성을 동시에 확보한다. 기본 공격 패턴과 스킬 발동은 FSM으로 안정적으로 유지하면서, "누구를 압박할 것인가", "진입할 것인가 후퇴할 것인가", "밀집한 플레이어를 어떻게 분리할 것인가" 같은 전략적 판단을 강화학습이 담당한다.

### 2-2. 강화학습 설계

**관측 공간** — 보스는 위치 정보뿐 아니라 플레이어의 전투 습관까지 관측한다.
- 보스/플레이어 2명의 위치, HP 비율, 상호 거리
- 플레이어 상태 정보 (시전 중, 패링 중 등)
- 최근 전투 데이터에서 추출한 행동 편향 점수

**행동 공간** — 이산 행동 선택
- 접근 / 후퇴 / 우회(거리 유지) / 특정 대상 압박 / 대기

**보상 함수** — 단순 피해량이 아닌 전투 흐름 전체의 유리함을 기준으로 설계
- 플레이어 피해 적중 / 처치 시 양의 보상
- 보스 피해 수신 시 음의 보상
- 스킬 범위 밖 장기 대기 시 음의 보상
- 전투 시간 경과에 따른 시간 페널티

### 2-3. 플레이어 행동 편향 분석

강화학습이 의미 있게 작동하려면 플레이어 행동의 다양성이 보장되어야 한다. 이를 위해 9종의 행동 편향을 정의하고 수치화하여 보스 AI의 관측값 및 스킬 선택 가중치에 반영하도록 설계하였다.

> 근접 선호 / 원거리 유지 / 공격 집중 / 생존 우선 / 패링 의존 / 로프 기동 / 스킬 중심 / 팀 밀착 / 팀 분산

예를 들어 패링을 자주 성공시키는 플레이어에게는 다단 히트나 지속 피해형 대응을 우선하고, 로프 활용 빈도가 높은 경우에는 이동 제한이나 위치 봉쇄형 대응을 강화한다.

### 2-4. 학습 환경 설계

실제 플레이어 데이터 대신 규칙 기반/Behavior Tree 기반 플레이어 봇을 여러 유형(근접형, 원거리형, 회피형, 패링형, 로프 활용형)으로 구성하여 보스를 반복 학습시키는 방향을 채택하였다. PPO 알고리즘은 정책 업데이트가 비교적 안정적이어서 보스 행동이 갑작스럽게 붕괴되지 않는 장점이 있다.

### 2-5. 전투 프레임워크

강화학습 적용의 전제 조건으로, 안정적인 전투 시스템을 자체 설계·구현하였다.

- **StatsManager 단일 진실 공급원**: 모든 스탯 변경(HP/Shield/상태이상 12종/버프 4종/디버프 4종)을 하나의 매니저가 전담
- **조립형 스킬 시스템**: 37종 부품을 조합하여 31종 스킬을 생성하는 모듈형 구조
- **Host-authoritative 네트워크**: 클라이언트는 입력 의도만 전송, 서버가 판정 확정 후 동기화
- **카운터 관계 기반 드래프트**: 플레이어 빌드 선택에 따라 보스가 카운터 스킬을 선택하는 전략적 상호작용

## 4. 기대 효과

1. **적응형 보스전 경험**: 강화학습을 통해 플레이어 행동에 실시간 대응하여 반복 플레이에서도 정답이 고정되지 않는 보스전 구현
2. **하이브리드 AI 설계 경험**: 규칙 기반의 안정성과 학습 기반의 유연성을 결합하는 실무적 AI 설계 역량 확보
3. **플레이어 행동 분석 기반 전투 설계**: 플레이 스타일 자체를 시스템적으로 해석하고 수치화하는 기획 경험
4. **재사용 가능한 프레임워크**: 전투 시스템, 보스 FSM, 행동 편향 분석 구조, 강화학습 연동 아키텍처를 향후 프로젝트에 응용 가능
5. **연구성과와 작품성 겸비**: 협동 보스전에서의 강화학습 활용이라는 연구적 가치와, 플레이어가 체감하는 전투 변화라는 작품적 가치를 동시에 추구
